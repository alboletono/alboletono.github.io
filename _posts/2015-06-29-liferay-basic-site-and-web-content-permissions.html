Site association

When you are authenticated on a private Liferay site you have to belong to the site. The default site has been created at the first run of the Liferay Portal.

Once created, you have to tell that all your authenticated users belong to this site by default. This can be set by the following portal properties :

#
# Input a list of default user group names separated by \n characters that
# are associated with newly created users.
#
admin.default.user.group.names=My site

#
# Set this to true to ensure that a user is synchronized with the default
# associations of groups, roles, and user groups upon every login. Set this
# to false if default associations should only be applied to a user when a
# user is created.
#
admin.sync.default.associations=true

 

If you do not set these properties all your new users won't be able to see your site pages. This will be notified as a 404 page not found by the portal (which is not really true, in fact this is more a "not allowed" error).

 

Web content permissions

By default, the web content will be seen by any user that can see the page.

To have a more fine grain approach you should set journal.article.view.permission.check.enabled to true.

Then you will have to define for each role the permissions "Web Content" and "Manage Web Content" to "see".
