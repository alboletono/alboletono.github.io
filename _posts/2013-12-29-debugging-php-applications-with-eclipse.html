I remember what I was using 5 years ago to debug PHP applications. It was not real debugging and only "output observation" under apache. I was using zend framework with a WAMP application and Eclipse.

Now, it is possible to do far much better. The only is that it needs some extra configuration that you should know (and that is widely described on the web). It is really a shame that there is no php.ini embedded into Eclipse (at least the main updates to perform).

By following the below sites instructions I made it possible to use Eclipse in a debug mode (like for Java). The main interest for me whas to get the output of my functions for testing purpose like phpunit. I didn't want to modify my script and then refresh my web page (it is a loss of time and end user testing).

Here are the valuable sites I found:

  • http://tech.rickumali.com/blog/2011/11/13/getting-php-debugging-xdebug-working-windows
  • http://www.thierryb.net/pdtwiki/index.php?title=Using_PDT_:_Installation_:_Installing_the_XDebug_Debugger

What I mainly did on my Debian distribution is:

  1. Install Eclipse with PDT
  2. Download and install xdebug on the official site
  3. Install apache, php-cli debian packages
  4. Edit my php.ini (see detailed configuration above)
  5. Edit my Eclipse PHP preferences (Debug and executables part with xdebug)

My php.ini configuration:

[code language="text"]

output_buffering = 0

implicit_flush = On

[xdebug]

xdebug.remote_enable=1

xdebug.remote_host="localhost"

xdebug.remote_port=9000

xdebug.remote_handler="dbgp"

xdebug.remote_log=c:eclipse-phpxdebug.log

[/code]
