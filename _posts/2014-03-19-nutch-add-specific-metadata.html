Here is the practical case: crawling a cooking site (marmiton.org).

Initially, I made the mistake of copy/pasting parser-html (yes, yes!) and update it according to my needs.

There is a better way (of course ;)). By exploring the code I noticed that there was a HtmlParseFilter interface! I've observed the "parse-js" code and see that it is doing something like I would.

So I've duplicated the code of parse-js and adapt it to fit my needs. I could have used directly regexes instead of node walkers but no matter for my exercise it is working.

I've added some JUnit tests to avoid deploying each time my extension to nutch and "voil√†!", it is working!

Do not forget to reference your new parser into your nuch-site.conf (in plugin-includes section).
